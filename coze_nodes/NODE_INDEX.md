## 基础输入输出节点

| 节点 | 节点作用 | 关键输入 | 关键输出 | 注意事项 |
|---|---|---|---|---|
| 开始 | 工作流起点，用于声明启动工作流需要的信息。 | 当前可见输出字段为 `input`，类型 `string`，必填。 | `input` | 后续做业务工作流时，建议把 `input` 改成明确字段，例如 `product_name`、`selling_points`、`image_urls`。 |
| 结束 | 工作流终点，用于返回最终结果。 | `output`，类型 `string`。 | 最终返回给调用方的结果。 | 结束节点应统一返回结构化 JSON 字符串，方便接口调用方解析。 |
| 输入 | 中间过程输入节点。 | 无上游入参；输出 schema 中有 `input`。 | `input`，类型 `string`，必填。 | 适合需要中途补充用户信息的对话流；纯 API 工作流尽量少用。 |
| 输出 | 中间消息输出节点，支持流式或非流式输出。 | `content`。 | 消息内容。 | 适合在长流程中提示进度；最终接口结果仍建议从结束节点返回。 |

## 流程控制节点

| 节点 | 节点作用 | 关键输入 | 关键输出 | 注意事项 |
|---|---|---|---|---|
| 选择器 | 根据条件选择一个分支运行。 | 分支条件。 | 运行命中的分支。 | 适合按图片类型、修图类型、是否上传图片、是否加 Logo 分流。 |
| 意图识别 | 识别用户输入意图并匹配预设意图。 | 用户输入，当前 prompt 可见为 `{{query}}`。 | `classificationId`，类型 `integer`。 | 当前 intents 为空，使用前必须补意图列表，例如“文生图、图生图、抠图、画质提升”。 |
| 循环 | 按次数或条件重复执行一组节点。 | 循环次数或循环数据。 | 循环体输出。 | 适合逐张图片处理，但要注意超时和并发限制。 |
| 批处理 | 对列表数据批量运行处理体。 | 图片 URL 列表、Prompt 列表等。 | 批处理结果集合。 | 适合批量文生图、批量图生图、批量抠图、批量画质提升。 |
| 变量聚合 | 汇总多个分支输出。 | 多个分支变量。 | 聚合后的变量。 | 适合选择器后统一返回格式。当前 mergeGroups 为空，需要按业务配置。 |
| 变量赋值 | 给可写变量赋值。 | 变量和值。 | `isSuccess`，类型 `boolean`。 | 适合保存中间状态；普通 API 工作流不宜滥用。 |
| 异步任务 | 触发任务中心预定义异步任务。 | 异步任务入参。 | `taskId`、`msg`。 | 适合耗时图片批处理；调用方需要根据 `taskId` 查询结果或等待回调。 |

## 知识库与长期记忆节点

| 节点 | 节点作用 | 关键输入 | 关键输出 | 注意事项 |
|---|---|---|---|---|
| 知识库写入 | 写入文本知识库。 | `datasetList`、文档内容等。 | `documentId`、`fileName`、`fileUrl`。 | 当前未绑定知识库。批图匠工作流通常不优先使用，除非要沉淀商品知识。 |
| 知识库检索 | 从知识库召回相关内容。 | `datasetList`、`topK`，当前 `topK=5`。 | `outputList[]`，含 `output`、`documentId`。 | 可用于从品牌规范、商品资料中提取卖点。当前未绑定知识库。 |
| 知识库删除 | 删除知识库文档。 | `datasetList`、文档 ID。 | `isSuccess`，类型 `boolean`。 | 涉及数据删除，除非用户明确要求，不要默认加入业务工作流。 |
| 长期记忆写入 | 写入长期记忆。 | `messageList`。 | `isSuccess`，类型 `boolean`。 | 原始内容是示例数据。批图匠项目里慎用，避免把用户业务数据误写入记忆。 |

## 图片生成与处理节点

| 节点 | 节点作用 | 关键输入 | 关键输出 | 注意事项 |
|---|---|---|---|---|
| 图像生成 | 根据文字描述或参考图生成图片。 | `prompt.prompt`、`prompt.negative_prompt`、`references`、模型设置。 | `data`、`msg`。 | 当前模型设置可见为 2048 x 2048、1K、`png`、`max_images=1`。适合文生图和图生图。 |
| 图像生成_1 | 第二个图像生成节点模板。 | 同“图像生成”。 | `data`、`msg`。 | 可作为多分支备用节点，不代表已经配置了不同业务逻辑。 |
| 画板 | 自定义画板排版，支持添加文本和图片。 | `canvasSchema`。 | `data`、`msg`。 | 当前画布为 1920 x 1080 白底空画布。适合后期排版、加文案、加 Logo。 |
| sd_better_prompt | 插件：提示词优化。 | `prompt`，类型 `string`，必填。 | `data`、`msg`，均为 `string`。 | 适合把中文商品卖点优化成更适合图片生成的 Prompt。 |
| cutout | 插件：抠图，保留图片前景主体。 | `url` 图片、`prompt`、`output_mode`、`only_mask`。 | `data` 透明背景图、`mask` 蒙版、`msg`。 | `output_mode`：0 透明背景图，1 蒙版矢量图。`only_mask`：0 抠图结果尺寸，3 原图尺寸。 |
| image_quality_improve | 插件：画质提升。 | `image_url`，类型图片，必填。 | `data` 图片、`msg`。 | 适合批量清晰度提升；可作为生成或修图后的增强步骤。 |
