# 批图匠项目级 Agent 指南

## 项目定位

本项目用于保存和测试已经创建好的“批图匠”生图 Prompt 工作流。

这里的核心工作流不是直接生成最终图片，而是根据商品、卖点或图片生成需求，生成可交给图片生成模型使用的结构化生图 Prompt、负面 Prompt、图片角色、画面策略和相关说明。

后续 Agent 的核心任务不是开发该网页本身，也不是默认从零设计工作流，而是：

- 保存、阅读和测试已创建好的生图 Prompt 工作流。
- 登记工作流的调用接口、`workflow_id`、Input Schema、Output Schema、调用示例和实测记录。
- 根据用户复制到 ChatGPT 网页端或提供给 Codex 的节点内容，整理工作流说明和测试结论。
- 必要时根据截图补充批图匠页面功能、字段含义和业务背景。
- 明确哪些内容来自已实测工作流，哪些只是节点素材或截图推断。

## 已知网页应用内容

从当前截图可确认，批图匠网页应用主要是一个批量 AI 图片生成和图片处理工具，左侧导航包含：

- 批量文生图
- 批量图生图
- 批量 AI 修图
- 批量模特换装
- 暂无生成任务
- 历史生成记录

顶部区域包含：

- 批图匠 ai 品牌标识
- 首页
- 使用教程
- 充值中心
- 金豆余额
- 张图额度
- 用户账号入口

右侧通常是历史生成结果区域，包含：

- 生成时间
- 本次任务的文字描述或处理指令
- 图片模型
- Logo 内容
- Logo 位置
- 图片尺寸比例
- 结果图片预览
- 重新编辑

- 全部下载
- 分页或回到底部按钮

## 页面能力理解

### 批量文生图

页面地址片段可能是 `#/batchTextToImage`。

该功能用于根据文字描述批量生成图片。

截图中可见的输入和配置包括：

- 图片类型标签：主图、套图、详情图、海报。
- 文本输入框：占位内容为“产品+卖点”。
- 每张图片 AI 生成数量。
- 模型选择，例如 `Ptu1.0`。
- 批量加文字/LOGO，可选。
- LOGO 位置、LOGO 名称或点击上传 LOGO。
- 图片尺寸，例如 1:1。

适合生成的 Prompt 方向：

- 电商商品主图 Prompt。
- 商品套图 Prompt。
- 商品详情图 Prompt。
- 商品海报 Prompt。
- 带 Logo 或品牌元素的图片 Prompt。

### 批量图生图

页面地址片段可能是 `#/batchImgToImg`。

该功能用于上传已有图片，并根据文字描述重新生成或优化图片。

截图中可见的输入和配置包括：

- 图片类型标签：主图、套图、详情图、海报。
- 图片上传区域，限制最多 10 张。
- 支持格式：jpeg、png、webp、bmp、tiff、gif。
- 图片大小限制：不超过 10MB。
- 图片宽高比限制：在 1/16 到 16/1 之间。
- 图片尺寸限制：不超过 10MB 总像素或 6000 x 6000 级别限制。
- 文本输入框：占位内容为“产品+卖点”。
- 每张图片 AI 生成数量。
- 模型选择，例如 `Ptu1.0`。

适合生成的 Prompt 方向：

- 保留原商品主体，重新排版优化。
- 不改变图片尺寸，重做电商主图。
- 基于已有图片生成套图、详情图或海报。
- 提升商品图清晰度、构图、商业感。

### 批量 AI 修图

页面地址片段可能是 `#/batchBgReplace`。

该功能用于批量上传图片后进行 AI 修图。

截图中可见的修图模式包括：

- 去水印
- 改文案
- 抠图

可见配置包括：

- 上传需要修改的图片，最多 10 张。
- 支持 jpeg、png、webp、bmp、tiff、gif。
- 文件不超过 10MB。
- 修图指令输入区。
- 每张图片 AI 生成数量。
- 模型选择，例如 `Ptu1.0`。
- 批量加文字/LOGO，可选。

适合生成的 Prompt 方向：

- 去除图中水印。
- 替换或优化图片中的文案。
- 抠出主体并生成透明背景或白底图。
- 按用户描述修改局部内容。

### 批量模特换装

页面地址片段可能是 `#/batchOutfitSwap`。

该功能用于把服装图片替换到模特图片上，生成模特穿搭图。

截图中可见的输入和配置包括：

- 图片类型标签：主图、套图、详情图、海报。
- 更换服装图上传区。
- 模特图片上传区。
- 每个上传区最多 10 张。
- 输入文字描述，可选。
- 每张图片 AI 生成数量。
- 模型选择，例如 `Ptu1.0`。
- 下方可能还有批量加文字/LOGO 等扩展配置。

适合生成的 Prompt 方向：

- 保持模特姿势和人物特征，把服装自然替换到模特身上。
- 生成电商服装主图、套图、详情图或海报。
- 控制服装版型、材质、颜色、Logo 和展示角度。

## Agent 工作方式

### 处理截图时

1. 先说明哪些内容是截图中明确可见的。
2. 再说明哪些内容是根据常见电商 AI 作图工具推断的。
3. 不要把推断内容写成确定事实。
4. 看不清的文字、按钮、字段，不要编造。
5. 如果字段含义会影响 Prompt 或 Coze 工作流，必须向用户确认。

### 生成 Prompt 时

Prompt 应优先面向“电商批量作图”场景，结构建议包含：

- 任务目标
- 输入图片或输入文本
- 商品主体
- 卖点信息
- 画面风格
- 构图要求
- 背景要求
- 尺寸比例
- Logo 或文字要求
- 禁止事项
- 输出要求

Prompt 要尽量可复制到 Coze 或图片生成节点中使用。

### 设计 Coze 工作流时

优先处理已经创建好的生图 Prompt 工作流。只有用户明确要求新增或重组流程时，才进入“设计 Coze 工作流”模式。

Coze 工作流方案应尽量写清楚：

- 触发入口
- 用户输入字段
- 文件上传字段
- 参数校验节点
- Prompt 拼接节点
- 图片生成或修图节点
- 结果整理节点
- 失败重试逻辑
- 输出给用户的内容

如果截图没有体现真实接口或节点名称，不要假装知道接口细节。应使用“图片生成节点”“图片处理节点”“结果汇总节点”等通用名称，并标注需要按实际 Coze 插件能力替换。

如果需求是根据已提供的扣子节点直接分析、测试或补充工作流，先查看 `coze_nodes/PATH_INDEX.md` 和 `coze_nodes/NODE_INDEX.md`：

- `PATH_INDEX.md` 用于选择文生图、图生图、抠图、画质提升、画板排版、意图识别分流等可复用路径。
- `NODE_INDEX.md` 用于确认节点用途、入参、出参和注意事项。
- `raw/` 中保留原始 Coze 剪贴板内容，只作为证据和细节参考；原始内容没有连线，不要误认为是已完成工作流。
- 如果用户说“我把节点复制到 ChatGPT 网页里操作”，应把这些节点视为 ChatGPT 网页端可读的工作流素材，帮助用户整理操作步骤、输入输出和测试结果。

### 登记 Coze 工作流时

所有已确认或待确认的 Coze 工作流，都统一写入 `WORKFLOWS.md`。当前优先维护的是“生成生图 Prompt”的工作流。

登记时必须包含：

- 工作流编号，例如 `WF-001`。
- 页面或功能归属。
- 工作流名称。
- Coze 调用接口。
- 请求方式。
- `workflow_id`。
- 鉴权变量名，例如 `${COZE_API_TOKEN}`。
- Input Schema。
- Output Schema。
- 配套说明图状态。
- 调用示例。
- 实测记录。
- Changelog。

对于“生成生图 Prompt”的工作流，登记和测试时要重点确认：

- 输入字段是否能表达商品名称、产品卖点、图片类型、尺寸比例、Logo 或文字要求。
- 输出字段是否包含可直接用于生图的正向 Prompt、负面 Prompt、图片角色、可见文案和画面策略。
- 返回内容是否需要二次 JSON 解析。
- 字段名是否和实际返回一致，不要私自修正实测字段名。

安全规则：

- 不要把 Coze Bearer Token、API Key 等敏感信息明文写入项目文件。
- 调用示例统一使用环境变量占位。
- Output Schema 以实际调用结果为准；没有实测结果时，必须标记为“待确认”。
- 每次新增、修改或补充实测结果，都要更新对应工作流的 Changelog。

如果用户提供工作流调用函数、Input Schema、Output Schema，并要求生成一张给开发同事看的说明图：

- 使用 ImageGen 工具生成图片。
- 图片目标是让开发同事一眼理解调用函数、输入、输出和数据流。
- 图片中只能展示环境变量名，不能展示明文 token、API Key 或其他敏感信息。
- 生成后要把图片路径或交付位置记录到 `WORKFLOWS.md` 对应工作流的“配套说明图”区域。
- 如果图片内容依赖未确认字段，先标记“待确认”，不要把推测内容画成确定事实。

- 如果需要根据截图继续完善，应先提出关键问题，而不是猜测。
- 可以更新本文件和已有 README。
